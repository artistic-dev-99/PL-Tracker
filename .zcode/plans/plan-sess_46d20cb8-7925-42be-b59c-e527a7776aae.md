# PL Tracker 2 — UI Polish Plan (Refine Apple/Glass, High-Impact Quick Wins, SVG Icons)

## Goal
Elevate the existing Apple-inspired glassmorphism UI to feel premium and consistent — by introducing a real **spacing scale**, **typography scale**, **elevation system**, **consolidated token usage**, **polished components**, and a **unified SVG icon language** (replacing emoji). Current look is preserved and refined, not replaced.

All work is in three files:
- `client/static/style.css` (2,295 lines)
- `client/static/index.html` (1,142 lines)
- `client/static/app.js` (only if any icon is injected via JS — will verify)

No backend, no build step, no new dependencies.

---

## Phase 1 — Design Token Foundation (the highest-leverage change)

This is the change that makes *every* component look more intentional. Add tokens at `style.css:2-55` (`:root`) and mirror in the dark-mode block (`style.css:58-81`).

**1.1 Spacing scale** (currently ad-hoc: 30/24/20/15/6 px)
Add to `:root`:
```css
--space-1: 4px;  --space-2: 8px;  --space-3: 12px;
--space-4: 16px; --space-5: 20px; --space-6: 24px;
--space-7: 32px; --space-8: 40px; --space-9: 56px;
```
Then sweep the most visible components to use them: `.glass-card padding: var(--space-7)`, `.app-sidebar padding: var(--space-6) var(--space-4)`, `.input-group gap/ margin: var(--space-3)/var(--space-4)`, `.form-grid gap: var(--space-6)`, `.stat-card padding: var(--space-5)`, `.app-content-header padding: var(--space-5) var(--space-7)`. (Search-and-replace pass; no layout changes.)

**1.2 Typography scale** (currently ad-hoc: 1.5/1.15/0.92/0.85/0.72 rem)
Add to `:root`:
```css
--text-xs: 0.72rem; --text-sm: 0.85rem; --text-base: 0.95rem;
--text-md: 1.1rem;  --text-lg: 1.25rem; --text-xl: 1.5rem; --text-2xl: 1.875rem;
--leading-tight: 1.15; --leading-normal: 1.5;
```
Apply: `h1` → `var(--text-xl)`, `h2` → `var(--text-lg)`, `.subtitle` → `var(--text-sm)`, `.label`/`th` → `var(--text-xs)`, `.logo-text` → `var(--text-md)`.

**1.3 Radius + elevation system** (radii exist; elevations are scattered)
Extend tokens:
```css
--radius-xl: 20px;
--elevation-1: 0 1px 2px rgba(0,0,0,.04);
--elevation-2: 0 4px 16px rgba(0,0,0,.06), 0 1px 3px rgba(0,0,0,.03);
--elevation-3: 0 12px 40px rgba(0,0,0,.08), 0 2px 6px rgba(0,0,0,.04);
```
Map `.glass-card` → `--elevation-3`, `.modal-card` → `--elevation-3`, hover states → `--elevation-2`, inputs → `--elevation-1` (inset).

**1.4 Fix hardcoded colors → tokens (fixes dark-mode breakage).** Audit-and-replace these specific values:
- `.logo-icon-svg stroke: #0066cc` (`style.css:275`) → `var(--accent)`
- `.logo-accent color: #0066cc` (`style.css:286, 502`) → `var(--accent)`
- `.user-avatar-circle border/color #0066cc` (`style.css:434,439`) → `var(--accent)`
- `.sidebar-toggle-btn:hover background: #f1f5f9` (`style.css:303`) → `var(--surface-hover)`
- `.connection-status background: #f1f5f9` (`style.css:511`) → `var(--subtle-bg)`
- `.stat-icon background: #eff6ff; border rgba(0,102,204,.1)` (`style.css:1694-1697`) → `var(--accent-tint)`
- `.search-btn color: #d07d00/#92400e` (`style.css:961-966`) → `var(--warning)` with opacity mixing
- `.fallback-bar-outer background: rgba(255,255,255,.05)` (`style.css:1787`) → `var(--subtle-bg)`

**1.5 Consistent input height.** Comments say `42px` (`adjustments_log.md` Phase 2.2) but CSS says `40px` (`style.css:707`). Standardize on `44px` (better touch target, modern macOS norm), applied to `input/select/textarea` base.

## Phase 2 — Component Polish (the visible wins)

**2.1 Buttons** (`style.css:873-967`)
- Base button: `height: 40px; padding: var(--space-3) var(--space-5); font-weight: 600; letter-spacing: -0.01em;`
- Primary: add subtle gradient overlay + `box-shadow: var(--elevation-1), 0 0 0 0 var(--accent-glow)` and on hover a glow ring `0 4px 14px var(--accent-glow)`.
- Add `.btn-lg` (height 48px) and `.btn-sm` (height 32px) modifiers for action bars.
- Make `.btn-icon` (square, padding 0) for sidebar/settings icon buttons.

**2.2 Inputs** (`style.css:662-740`)
- Increase focus ring: `border-color: var(--accent); box-shadow: 0 0 0 4px var(--accent-glow);` (a halo, not just a border).
- Floating-label optional skip — keep current labels but tighten: label→`var(--text-xs)`, `text-transform: uppercase; letter-spacing: 0.04em` to match `th` style for consistency.
- Add `transition: border-color .15s, box-shadow .15s`.

**2.3 Glass cards** (`style.css:173-200`)
- Add `backdrop-filter: blur(24px) saturate(180%)` for richer frosted feel.
- Unify hover: gentle border-color shift + `--elevation-3` lift on interactable cards (stat cards, chart cards). Non-interactive cards (login) stay static.

**2.4 Tables** (`style.css:1293-1331`)
- Increase `th` padding to `var(--space-4) var(--space-4)`, make header row slightly tinted (`background: var(--subtle-bg)`).
- `td` padding → `var(--space-3) var(--space-4)`.
- Replace zebra stripe with cleaner alternating row: `tr:nth-child(even) td { background: var(--subtle-bg); }`.
- Hover: add left accent bar `box-shadow: inset 3px 0 0 var(--accent)`.
- Add `border-radius: var(--radius-md); overflow: hidden` on table container.

**2.5 Stat cards** (`style.css:1685-1720`, `index.html:544-565`)
- Bigger number: `--text-2xl`, tighter `letter-spacing: -0.03em`.
- Add subtle top accent gradient bar per card (color-coded: blue/green/orange/purple).
- Replace emoji `.stat-icon` with inline SVG (see Phase 3).

**2.6 Sidebar** (`style.css:234-455`)
- Active tab: add left accent indicator `box-shadow: inset 3px 0 0 var(--accent)` and slightly stronger `--accent-tint`.
- Add `font-weight: 600` already exists; tighten icon-to-text gap to `var(--space-3)`.
- Add active-tab icon color = `var(--accent)`.

## Phase 4 — Alignment & Final Polish Pass

**4.1 Vertical rhythm & alignment**
- Audit form grids (`.form-grid-3/-2`, `style.css:624-642`): ensure all use `gap: var(--space-6)` consistently (currently mix of 20/30).
- Standardize card heading spacing: add a `.card-header { margin-bottom: var(--space-5) }` utility, remove ad-hoc margins.
- Align action button rows: wrap submit/update/delete in `.form-actions` with `gap: var(--space-3)` and consistent `justify-content` (currently `mt-4` ad-hoc).

**4.2 Settings layout** (`style.css:1865`, sub-sidebar `1919`)
- Sub-sidebar tabs get same active-state treatment as main sidebar (accent left bar + tint).
- Sub-tab icon containers match `.tab-icon` (20px) so they don't feel larger/smaller than main nav.

**4.3 Modals** (`style.css:1339`)
- Increase `border-radius: var(--radius-xl)`.
- Modal header/body/footer rhythm: header `padding: var(--space-6)`, body `var(--space-6)`, footer border-top + `var(--space-4) var(--space-6)`.

**4.4 Login screen polish** (`index.html:85-128`, animated shapes)
- Tighten logo lockup: SVG in `.login-logo-box` aligned to brand title baseline; balance `gap: var(--space-4)`.
- Ensure "Sign In" button uses new `.btn-lg` style.

**4.5 Remove inline styles**
- Audit `index.html` for `style="..."` one-offs (noted in Settings small inputs) → move to reusable utility classes / component classes.

## Execution Order & Verification
1. **Phase 1 first** (tokens + color fixes) — visual lift across whole app immediately, low regression risk. Verify by toggling dark mode on every screen.
2. **Phase 2** (components) — apply per-component; check each tab after each.
3. **Phase 3** (SVG icons) — replace + verify each icon renders in both themes.
4. **Phase 4** (alignment pass) — final consistency sweep.

**Verification method:** Run the app (`server/server.py` + `client/client.py`), log in, walk every screen in **both** light and dark mode, confirm: no hardcoded-color breakage in dark mode, inputs are uniform height, all icons are SVG (no emoji), spacing feels consistent, tables/cards/buttons look elevated and aligned.

## Risk
Low. All changes are CSS variables and markup within 3 static files. No data layer, routing, or API touched. The token approach means a mis-step is a single-line fix in `:root`.
